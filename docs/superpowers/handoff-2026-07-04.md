## Invariants learned this round (don't relearn)

- **Audio MCLK/LRCK lock** (`facts.md` → "Audio: lock LRCK to MCLK/256"): MCLK is an
  integer PWM divide of the 250 MHz clock = 4.0984 MHz (not 4.096), so the MCLK-direct
  codec runs fs=16009 Hz. LRCK must be locked to MCLK/256, else the DAC slips ~9
  samples/s → an audible ~9 Hz tick. **Applies to any future clock-derived audio (PDM).**
- Harvested RX/DMA drivers often use `irq_set_exclusive_handler(DMA_IRQ_0,…)` — must be
  changed to `irq_add_shared_handler` (the ST7796 flush shares that line).
- Keep the repo's proven 250 MHz `board.c`; good drivers derive clocks from
  `clock_get_hz(clk_sys)` (don't adopt a source repo's different sysclk, e.g. 153.6 MHz).
