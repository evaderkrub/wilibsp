# PDM microphones — on-hardware E2E findings (2026-07-04)

**Increment 2 (PDM mics), branch `feat/pdm-mics`.**
Flashed `apps/hello_mics` (built from commit a839d14) to the FreeWili 2
(RP2350 rev 3, CMSIS-DAP probe serial E664B495075C2A2F, `fw flash hello_mics`
programmed + verified OK) and streamed `fw rtt` for ~22 s.

## Observed RTT output

Boot sequence, exactly as designed:

```
ioexp: init ok (LCD reset released; CC1101 + 433 MHz antenna default)
=== hello_mics: 4-channel PDM microphone smoke test ===
physical mic order left->right: D, B, A, C
ioexp: MIC_PWR (P1_7) -> 1
pdm: init pio1 sm=0 dma=0 (4 mics, 16000 Hz PCM)
```

Then continuous per-mic lines ~3×/s, e.g.:

```
mic A: rms=    7 peak=   58   mic B: rms=    7 peak=   56   mic C: rms=    8 peak=   62   mic D: rms=    8 peak=   59
...
mic A: rms=    2 peak=    9   mic B: rms=    2 peak=   10   mic C: rms=    2 peak=   10   mic D: rms=    2 peak=   10
...
mic A: rms=    4 peak=   18   mic B: rms=    5 peak=   21   mic C: rms=    5 peak=   23   mic D: rms=    5 peak=   25
```

## Pass criteria

1. **All 4 channels alive with low ambient at rest — PASS.** Quiet-room floor
   is rms=2, peak 8–13 counts on every channel (consistent with the source
   repo's "mic PCM is ~±14 counts" idle observation). No channel stuck at 0,
   frozen, or pegged; values fluctuate sample-to-sample.
2. **Per-mic tap localization — PENDING (needs a human).** This run was
   unattended; nobody could tap each mic position. Strong indirect evidence
   the channels carry real acoustic signal: room-noise events (including the
   burst right after reset, rms 7–8 / peak 58–78, and a later bump to
   rms 4–5 / peak 18–25) hit **all four channels simultaneously and
   proportionally**, exactly as broadband room sound should — electrical
   noise or a stuck line would not produce correlated, transient,
   recovering activity across four independent CIC chains.
3. **Recovery to ambient after stimulus — PASS.** Both observed noise events
   decayed back to the rms=2 floor within one or two 100 ms blocks.

## Verdict

**Driver chain verified end-to-end on hardware:** ioexp mic power-on (P1_7),
pio1 SM claim, free-running ring DMA, CIC decimation to 16 kHz ×4, DC-block +
RMS in the app — all live and producing sane, dynamic, per-channel audio
levels. The only unchecked box is human tap localization per mic (criterion
2); to complete it, run `fw rtt` with `hello_mics` flashed and tap each mic
left→right — expect the spike order **D, B, A, C**.

No bugs found; no code changes were needed during hardware verification
(unlike increment 3, which surfaced the SPI1-init bug).
